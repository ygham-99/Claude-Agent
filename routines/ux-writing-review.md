## 실행 순서

### STEP 1: 검수 대상 이슈 수집

Jira에서 아래 JQL로 이슈를 조회하세요:
```
assignee = "712020:4b554afe-a990-49ed-afc9-bcdc1797eb9f"
AND status in ("대기", "진행중")
AND updated >= -12h
```

### STEP 2: 각 이슈의 댓글에서 Figma 링크 추출

각 이슈의 댓글 목록을 확인하고:
- `figma.com` URL이 포함된 댓글이 있는 이슈만 처리합니다.
- 댓글 중 `[UX 라이팅 검수]` 텍스트가 이미 있으면 **스킵**합니다 (중복 방지).

### STEP 3: Figma 파일에서 텍스트 추출

Figma MCP의 `get_design_context` 도구로 해당 파일의 텍스트 노드를 추출합니다.
- 버튼 레이블, 에러 메시지, 토스트, 플레이스홀더, 제목, 본문 등 모든 UI 문구 대상
- 텍스트가 없거나 추출 실패 시 해당 링크는 스킵

### STEP 4: UX 라이팅 원칙으로 검수

아래 **비즈하우스 UX 라이팅 가이드**를 기준으로 각 문구를 검토합니다.

---

#### [0] 브랜드 성격
- 친절하고 상냥하며 적극적인 태도
- 긍정적이고 해결 중심적 태도
- 제약 안내 시 이유 + 해결방법 함께 제시
  - ❌ "인쇄 품질을 위해 최소 폰트 사이즈를 6 이상으로 제약합니다."
  - ✅ "선명한 인쇄를 위해 글씨 크기는 6pt 이상으로 디자인해주세요."

#### [1] 일관성
- **1.1** 국립국어원 맞춤법 준수 (띄어쓰기, 문장 부호)
- **1.2** 동일한 용어 사용 (유사 의미 단어 통일)
- **1.3** '-해요'체 사용 (-해요, -해주세요, -하세요, -아요/어요)
- **1.4** 표기법:
  - 숫자: 3자리마다 콤마 (15,000원)
  - 단위: 숫자 뒤 붙여쓰기 (5,128개), 문자 뒤 띄어쓰기 (6만 2천 개)
  - 날짜: YYYY.MM.DD.
  - 가격: `~`와 `부터` 동시 사용 금지

#### [2] 간결성
- **2.1** 문장부호 남용 금지 (느낌표 1개, 이모지 문단당 1개, 특수문자 이모티콘 금지)
- **2.2** 동의어·유의어 중복 지양
- **2.3** 의미 없는 단어 삭제 (부사, 수식어 최소화)
- **2.4** 버튼에 '-하기'체 지양
  - ❌ 취소하기, 삭제하기, 확인하기
  - ✅ 취소, 삭제, 확인
  - 단, 주문/결제/목적지향 단일 CTA는 예외

#### [3] 보편성
- **3.1.1** 성별·연령 특정 금지
- **3.1.2** 디바이스 중립 언어 (터치하다→누르다, 스와이프→넘기다, 모니터→화면)
- **3.1.3** 지나친 은어·밈 지양
- **3.2.1** 한자어는 구어체로 변환
- **3.2.2** 전문용어에 추가 설명 병기 (예: 오시선(접힘선))

---

### STEP 5: 결과를 Jira 코멘트로 작성

검수 결과를 해당 Jira 이슈에 코멘트로 추가합니다.

#### 위반 사항이 있는 경우:
```
[UX 라이팅 검수] 🔍 자동 검수 결과

검수 대상 Figma: {Figma URL}

**위반 항목**
| 원칙 | 현재 문구 | 위반 내용 | 수정 제안 |
|------|-----------|-----------|-----------|
| [2.4] 버튼 '-하기'체 | "확인하기" | 불필요한 '-하기'체 | "확인" |
| [1.3] '-해요'체 | "입력하십시오" | 딱딱한 어미 | "입력해주세요" |

> 🤖 이 검수는 자동으로 생성되었습니다. 최종 판단은 담당 PD에게 있습니다.
```

#### 위반 사항이 없는 경우:
```
[UX 라이팅 검수] ✅ 자동 검수 결과

검수 대상 Figma: {Figma URL}

UX 라이팅 원칙 위반 항목이 발견되지 않았습니다.

> 🤖 이 검수는 자동으로 생성되었습니다. 최종 판단은 담당 PD에게 있습니다.
```

## 주의사항
- Figma 링크가 없는 이슈는 처리하지 않습니다.
- 이미 `[UX 라이팅 검수]` 코멘트가 달린 이슈는 스킵합니다.
- 텍스트 추출 실패 시 코멘트에 실패 사유를 남깁니다.
- 검수 결과는 참고용이며, 최종 판단은 담당 PD의 몫입니다.
